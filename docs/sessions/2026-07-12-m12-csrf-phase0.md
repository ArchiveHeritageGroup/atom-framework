# M12 CSRF - Phase 0 deployed (Symfony-path enforcement, log mode) - 2026-07-12

**Released:** atom-framework (this release). **Status:** Phase 0 live on PSIS + archaeology in **`log` mode** (validate + record violations, never block). Phases 1-3 (observe -> tokenize forms -> flip to `enforce`) are operational, to run over real traffic.

## What shipped

1. **Latent bug fixed - `CsrfService::getEnforcementMode()` always returned `enforce`.**
   `CsrfService` (namespace `AtomFramework\Services`) referenced `AhgSettingsService::class` without importing it, so it resolved to a non-existent `AtomFramework\Services\AhgSettingsService`; `class_exists()` was always false and the method fell through to the hardcoded `'enforce'` default. The `csrf_enforcement` setting was therefore **never honoured**. Fixed to the fully-qualified `\AtomExtensions\Services\AhgSettingsService`. This was discovered because the first activation blocked (403) despite the setting being `log`; after the fix the same request returns 200.

2. **Symfony-path wiring - `AhgController::preExecute()`** now calls `enforceCsrf()` (guarded by `method_exists`), mirroring the standalone `dispatch()` path. Previously mutating AhgController actions served through `index.php` had no CSRF protection.

3. **Violation logging to two sinks** (the PSIS `www` php-fpm pool discarded raw `error_log()` output):
   - **`ahg_error_log` table** - `CsrfService::enforce()` inserts a `level=warning`, `exception_class=CsrfViolation` row (url, method, ip, ua, hostname, status_code=403 only in enforce mode). Viewable at `/ahgSettings/errorLog`. This is the primary Phase-1 harvest surface.
   - **`www` pool file** - `/etc/php/8.3/fpm/pool.d/www.conf` now sets `php_admin_value[error_log] = /var/log/php-psis-error.log` + `log_errors = on`.

## Per-server state (NOT in git - set on each host)

- `ahg_settings.csrf_enforcement = 'log'` seeded on both the `archive` and `archeology` databases (group `security`).
- `www.conf` pool `error_log` directive (above) + `/var/log/php-psis-error.log` (owned www-data).

## Verified

- PSIS homepage 302; tokenless POST to an AhgController route (`/customFields/save`) returns **200** (log mode, non-blocking).
- Same request records to **both** sinks: `ahg_error_log` row (`CSRF token missing (mode: log)`) and the pool file (`CSRF violation: POST ... mode: log`).
- Regression proof: before the CsrfService fix the same POST was **403** (enforce); after, **200** (log).

## Phase 1-3 (remaining, operational)

1. **Observe** - let real traffic run in `log` mode; harvest distinct `METHOD URL` from `ahg_error_log` (exception_class `CsrfViolation`). Each is a form/AJAX endpoint missing a token.
2. **Tokenize** - add `CsrfService::renderHiddenField()` / `getMetaTag()` (+ `X-CSRF-TOKEN` header for AJAX) to the endpoints the log names. Locked plugins that appear need explicit unlock.
3. **Flip** - set `ahg_settings.csrf_enforcement = 'enforce'` once the log is quiet. Instant rollback: set back to `log` (or `off`).
